### 🧠 &nbsp; How I work

- **Boring tech first.** I won't introduce a framework that costs more than the problem.
- **Real dependencies in tests.** Testcontainers over mocks, every time.
- **Idempotency from day one.** Money endpoints and event consumers ship with keys.
- **Compile-time over runtime.** Branded types, discriminated unions, strict TS.
- **One log per transition, not per line.** Structured events; metrics for counts; traces for latency.
